# Политика безопасности / Security Policy

## Поддерживаемые версии / Supported Versions

Ниже указано, какие версии **mihomo-proxy-ros** получают обновления безопасности:

| Version         | Supported          |
| --------------- | ----------------- |
| main / latest   | :white_check_mark: |

> 🔹 **main / latest** — это ветка с последними изменениями и багфиксами, рекомендуемая для использования.  
> 🔹 Старые версии больше не поддерживаются и могут содержать уязвимости.

---

## Сообщение о уязвимости / Reporting a Vulnerability

Если вы обнаружили проблему безопасности в **mihomo-proxy-ros**, пожалуйста:

1. Создайте **issue с тегом `security`** в GitHub репозитории.  
2. Или отправьте электронное письмо автору проекта:  
   **medium1992 [at] github.com**  

Мы гарантируем:

- Подтверждение получения сообщения в течение **48 часов**.  
- Оценку уязвимости и план действий в течение **7 дней**.  
- Сообщение о закрытии или исправлении уязвимости до её публичного раскрытия.  

> ⚠️ Не публикуйте уязвимость публично до её исправления — это может поставить пользователей под угрозу.

---

## Дополнительно

- Обновляйте Docker-образы и скрипты MikroTik, как только исправления безопасности будут выпущены.  
- Все новые Pull Request должны проходить проверку безопасности перед слиянием в `main`.
